Operating Pattern

반복 가능한 데이터 프로젝트 흐름

  1. 01 Collect

    API와 공개 데이터를 수집하고, 재현 가능한 설정과 수집 기준을 남김

  2. 02 Structure

    원천 row를 분석 질문에 맞는 mart, signature, 집계 단위로 변환

  3. 03 Model

    EDA 결과를 모델링으로 검증하고 누수, 표본, 기준선 문제를 점검

  4. 04 Ship

    결과를 대시보드, 서비스, 보고서로 연결해 실제 사용 가능한 산출물로 배포
